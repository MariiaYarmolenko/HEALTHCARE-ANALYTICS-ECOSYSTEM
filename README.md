# 🏥 Healthcare Analytics Ecosystem: Insights & Recommendations

## 📋 Огляд проєкту
Цей проєкт присвячений комплексному аналізу діяльності багатопрофільної клініки в штаті Массачусетс. Основна мета — вивчення пацієнтопотоку, операційної ефективності та клінічних профілів для покращення прийняття управлінських рішень для оптимізації роботи клініки та зміцнення її фінансової стійкості. Аналіз охоплює дані за період з 2011 по початок 2022 року.

---

## 🏗 Data Architecture & Modeling
Для забезпечення високої продуктивності та цілісності аналізу було розроблено схему даних (ERD), яка об'єднує ключові сутності госпіталю:

* **Центральна сутність:** `Encounters` — містить дані про час візиту, клас допомоги та фінансові показники (Claim Costs).
* **Довідники:** `Patients` (демографія) та `Payers` (страхові компанії).
* **Деталізація:** `Procedures` — зв'язок "один до багатьох" для аналізу медичних маніпуляцій.

## 🛠 Технічний стек
- **SQL (MySQL Workbench):**
  - Створення **індексів** (`CREATE INDEX`) для прискорення операцій об'єднання (Joins) та фільтрації, що значно скоротило час виконання запитів на великих обсягах даних.
  - Розробка фінальної **вітрини даних (Data Mart)** шляхом агрегації даних про візити, процедури та білінг, що забезпечило миттєве завантаження візуалізацій у Tableau.
  - Використання **Temporary Tables** для декомпозиції складних розрахунків.
  - **Window Functions** (`LAG`, `OVER`) для аналізу повторних госпіталізацій (Readmissions).
  - Складні логічні конструкції (`CASE WHEN`) для трансформації сирих кодів діагнозів у зрозумілі бізнес-категорії.
- **Tableau:** розробка інтерактивної системи дашбордів та складних бізнес-метрик:
  - **LOD Expressions (FIXED):** розрахунок кількості візитів на одного пацієнта та конверсії з Emergency у стаціонар.
  - **Calculated Fields:** створення метрик Out-of-Pocket Cost, Long Stay Rate (%) та сегментації пацієнтів за віковими групами.
  - **Data Densification:** аналіз часових рядів для виявлення нічних піків навантаження.

## 📈 Ключові інсайти

### 1. Фінансовий ландшафт
* **Загальний обсяг претензій** склав $101M (Total Claim Cost).
* Понад 48% від усіх виставлених рахунків ($49M) припадає на категорію **NO_INSURANCE**, що вказує на значний обсяг потенційних безнадійних боргів та потребу в посиленому фінансовому супроводі.

### 2. Пріоритетний сегмент: Seniors (60+)
* Пацієнти вікової групи 60+ формують **понад 60% загального обсягу Claim Costs**.
* Висока вартість зумовлена медичною складністю випадків (діаліз, кардіомоніторинг) та необхідністю тривалого перебування у стаціонарі.

### 3. Операційна гнучкість
* Виявлено стабільні сплески госпіталізацій у нічний час, що вимагають перегляду графіків чергувань та ресурсної підтримки 24/7.

## 🚀 Strategic Growth Roadmap

### Етап 1: Операційна синхронізація (1–3 місяці)
- Адаптація штатного розкладу під реальні цикли завантаженості.
- Розвиток **Smart Triage** для підтримки цільової конверсії з Emergency у стаціонар на рівні 3.58%.

### Етап 2: Спеціалізація на Geriatric Care (3–9 місяців)
- Розширення відділень діалізу та кардіології для забезпечення потреб сегмента 60+.
- Запуск служби допомоги для незастрахованих пацієнтів з метою їх переведення на доступні державні програми.

### Етап 3: Цифрова трансформація та gрофілактика (9+ місяців)
- Запуск мобільного додатка для прозорого відображення **Claim Cost** та превентивних Wellness-нагадувань.
- Масштабування програм скринінгу для груп із високою вартістю лікування для переходу до моделі ранньої діагностики.
